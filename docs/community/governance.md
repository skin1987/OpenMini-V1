# OpenMini Community Governance

## 行为准则

### 我们的承诺

- 营造开放和欢迎的环境
- 尊重不同观点和经验
- 以建设性的方式接受批评
- 关注对社区的最佳利益
- 对其他社区成员表示同理心

### 不可接受的行为

- 骚扰、歧视或侮辱性言论
- 发布他人私人信息
- 其他不专业或不道德行为

## 决策流程

### RFC (Request for Comments)

重大变更需要经过 RFC 流程：

1. 在 `docs/rfcs/` 提交提案
2. 社区讨论 (至少 2 周)
3. Maintainer 审批
4. 实施与 Review

### 维护者职责

| 角色 | 职责 |
|------|------|
| **Core Maintainer** | 代码合并、Release 管理、架构决策 |
| **Maintainer** | PR Review、Issue 分类、文档审核 |
| **Contributor** | 提交代码、Bug修复、文档改进 |

### 成为 Maintainer

1. 持续高质量贡献 (3个月+)
2. 完成 10+ 个有意义的 PR
3. 参与 Code Review
4. 被 Core Maintainer 邀请

## Release 周期

| 类型 | 频率 | 说明 |
|------|------|------|
| **Patch** | 按需 | Bug修复, 安全补丁 |
| **Minor** | 月度 | 新功能, 兼容变更 |
| **Major** | 季度 | 架构重构, 不兼容变更 |

### 版本号规则

遵循 [SemVer](https://semver.org/)：
- **MAJOR**: 不兼容的API变更
- **MINOR**: 向后兼容的功能新增
- **PATCH**: 向后兼容的问题修复

## 社区活动

### 定期活动

- **每周三**: Office Hour (线上答疑)
- **每月第一个周五**: Community Call (路线图同步)
- **每季度**: Hackathon (48小时)

# Metal GPU 后端优化规范

## 完成状态: ✅ 生产就绪

## 已修复的问题

| 问题 | 修复方式 | 状态 |
|------|----------|------|
| Attention kernel 共享内存竞争 | 改用在线 Softmax，完全消除 threadgroup 内存依赖 | ✅ |
| 矩阵乘法 Grid 大小计算错误 | 使用 `(m + BLOCK-1)/BLOCK` 作为线程组数量 | ✅ |
| ShaderLibrary 锁粒度过大 | 改用 `RwLock` + 双重检查，支持并发读取 | ✅ |
| Kernel 内固定数组越界 | 移除 `scores` 数组，改为寄存器累加 | ✅ |
| bfloat16 特性检测不准确 | 基于 `supports_family(Apple7)` 判断 | ✅ |
| Mask 导致 NaN | 添加 `score > -1e30f` 跳过被屏蔽位置，并防止除零 | ✅ |
| 基准测试干扰单元测试 | 改为 `#[cfg(test)]` | ✅ |
| Threadgroup 内存声明位置 | 移到循环外，提高可读性 | ✅ |

## 延迟优化（非致命）

| 问题 | 状态 | 说明 |
|------|------|------|
| 线程组大小缩放逻辑 | ⏸️ 延迟 | 当前环境不会触发 (16x16=256 < 1024) |
| 同步等待限制吞吐量 | ❌ 未处理 | 可选优化，需要异步 API 设计 |

## 性能与正确性评估

- **正确性**: 所有单元测试通过，Attention 在带 mask 场景下不会产生 NaN
- **性能**: 矩阵乘法使用分块优化，Attention 使用在线 Softmax O(kv_len * head_dim)
- **并发安全**: ShaderLibrary 支持并发读取

## 结论

当前 `metal.rs` 文件已达到**生产就绪水平**。剩余问题均为工程优化项，不影响核心功能。
